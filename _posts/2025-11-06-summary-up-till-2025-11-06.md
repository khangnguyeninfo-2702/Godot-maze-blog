What I learn:
- First, I learn how to use 'TextureButton' nodes.
- Then I learn about the containers, like 'CenterContainer', 'VBoxContainer' or 'HBoxContainer' which are just to contains buttons in a specific order and layout.
- I also learn how to crop the sprite and divide them into suitable buttons' images in different states like hovering, pressed.
- Some physics with the background is also necessary, so I had to relearn some basic physics like air resistance and collisions.
---
First problem encountered:
- The spritesheet I downloaded didn't have a button like "start" or "play" that I needed for the main menu, this is when I use paint to separate the letters and just stick them onto different already existing blank buttons.
Second Problem encountered:
- Locating the buttons/game titles in the place I wanted them to be.
- Containers can help with the layout but they also limits my buttons' size when I first use them. 
- Also, I couldn't locate my buttons around the screen freely, I can only locate the whole container which is in another container and so everything moves uniformly which wasn't what I needed.
- In order to solve the button's size problem, I messed around with the godot's documentation and found out that I can actually change my button's size using the mode "scale" and adjust the mimimum x, mimimum y, components of the button for it to scale accordingly.
- And in order to solve the buttons' location problem, I created multiple containers containing containers and then I could actually move them around the way I wanted to. This results in a more complex structure of containers and buttons also but it's manageable.
Third problem encountered:
- When implementing the background, the shapes wasn't spawning randomly on the screen, instead they get packed in a corner at (0, 0) coordinate.
- To fix this, I used call deferred and set the block's position **before** adding them as a child of the background and then the blocks started spawning at random positions just like intended.
